Why Conventional Commits?

The Conventional Commits specification is a lightweight convention on top of commit messages. It provides an easy set of rules for creating an explicit commit history, making it easier to write automated tools on top of it.

Commit Structure


<type>(<scope>): <subject>

<body>

<footer>

Common Types

  • feat: A new feature.
  • fix: A bug fix.
  • docs: Documentation only changes.
  • style: Changes that do not affect the meaning of the code.
  • refactor: A code change that neither fixes a bug nor adds a feature.
  • test: Adding missing tests or correcting existing tests.
  • chore: Maintenance tasks (updating packages, CI/CD config).

Practical Example

feat(auth): add two-factor authentication
Implement TOTP-based F2A for user login security.
