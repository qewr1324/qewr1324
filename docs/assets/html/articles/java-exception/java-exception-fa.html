درک سلسله‌مراتب استثناها در جاوا

در جاوا، یک Exception (استثنا) رویدادی است که جریان عادی برنامه را مختل می‌کند. تمام انواع استثناها زیرکلاس‌های کلاس Throwable هستند. این سلسله‌مراتب به دو شاخه‌ی اصلی تقسیم می‌شود: استثناهای بررسی‌شده (Checked) و استثناهای بررسی‌نشده (Unchecked).

۱. استثناهای بررسی‌شده (استثناهای زمان کامپایل)

این استثناها آنهایی هستند که کامپایلر شما را مجبور می‌کند آن‌ها را مدیریت کنید (با استفاده از try-catch یا throws). آن‌ها نشان‌دهنده‌ی خطاهای خارجی هستند که یک برنامه‌ی خوب باید آن‌ها را پیش‌بینی کرده و از آن‌ها بهبود یابد.

  • SQLException: خطاهای دسترسی به پایگاه داده (قطع ارتباط، اشتباه در کوئری).
  • IOException: شکست‌های عمومی ورودی/خروجی (مثلاً پر شدن دیسک، قطعی شبکه).
  • FileNotFoundException: تلاش برای باز کردن فایلی که وجود ندارد.
  • ClassNotFoundException: ماشین جاوا (JVM) نمی‌تواند کلاس خاصی را در زمان اجرا پیدا کند.
  • InterruptedException: یک نخ (Thread) در حال خواب یا انتظار، قطع شده است.
  • ParseException: خطا در طول فرآیند تجزیه داده (مثلاً تبدیل رشته به تاریخ).

۲. استثناهای بررسی‌نشده (استثناهای زمان اجرا)

این موارد از RuntimeException ارث‌بری می‌کنند. کامپایلر شما را مجبور به مدیریت آن‌ها نمی‌کند. آن‌ها معمولاً نشان‌دهنده‌ی باگ‌های برنامه‌نویسی هستند، مانند خطاهای منطقی یا استفاده‌ی نادرست از یک API. با رعایت اصول کدنویسی بهتر می‌توان از آن‌ها جلوگیری کرد.

  • NullPointerException: دسترسی به یک متد یا فیلد روی یک شیء null.
  • ArithmeticException: تقسیم یک عدد صحیح بر صفر.
  • ArrayIndexOutOfBoundsException: دسترسی به آرایه با اندیس نامعتبر.
  • IllegalArgumentException: ارسال یک آرگومان نامناسب به یک متد.
  • NumberFormatException: تلاش برای تبدیل یک رشته‌ی غیرعددی به عدد.
  • ClassCastException: تلاش برای تبدیل یک شیء به کلاسی که به آن تعلق ندارد.
  • ConcurrentModificationException: تغییر یک مجموعه (Collection) در حین پیمایش آن.
  • IllegalStateException: فراخوانی یک متد زمانی که شیء در وضعیت نامعتبر است.

بهترین روش‌ها (Best Practices)

  • مدیریت استثناهای بررسی‌شده: همیشه آن‌ها را به‌خوبی مدیریت کنید تا از کرش کردن برنامه جلوگیری شود.
  • پیشگیری از استثناهای بررسی‌نشده: از بررسی‌های تدافعی استفاده کنید (مثلاً if (object != null)).
  • خاص‌گرایی به جای عمومی: استثناهای خاص (مثل IOException) را بگیرید، نه استثنای عمومی Exception را.
  • ثبت و انتشار مجدد: در لایه‌های سرویس، استثنا را لاگ کنید و اگر نیاز بود، یک استثنای تجاری سفارشی منتشر کنید.

مثال عملی

try {
    FileReader file = new FileReader("data.txt");
    BufferedReader br = new BufferedReader(file);
    String line = br.readLine();
} catch (FileNotFoundException e) {
    System.err.println("فایل پیدا نشد! در حال ایجاد یک فایل پیش‌فرض...");
} catch (IOException e) {
    e.printStackTrace();
}
